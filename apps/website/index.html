vibeste.rs

A quieter lobste.rs. Hide the vibe, mute the threads.

What it does

Hides the tag

The vibecoding tag is hidden wherever it annotates a story — home, story, and tag pages. Your Filtered Tags settings page is never touched.

Mutes the threads

Any comment mentioning vibecoding is collapsed into a “muted conversation thread” link. Click it to read the thread anyway.

Your rules

Choose which tags to hide, which words to mute, and whether to mute a whole reply tree or just the matching comment.

Install

The extension works on Chrome, Firefox, and Safari from a single code base. While store listings are pending, you can build and load it yourself: 

git clone https://github.com/norbauer/vibeste.rs
cd vibeste.rs
nix develop            # or: corepack enable pnpm
pnpm install
pnpm build             # Chrome → apps/extension/.output/chrome-mv3
pnpm build:firefox     # Firefox → apps/extension/.output/firefox-mv2
  • Chrome / Edge: open chrome://extensions, enable Developer mode, and “Load unpacked” the chrome-mv3 folder.
  • Firefox: open about:debugging → This Firefox → “Load Temporary Add-on”, and pick any file in firefox-mv2. (Or run pnpm dev:firefox.)
  • Safari: build with pnpm build:safari, then wrap it with Apple’s xcrun safari-web-extension-converter — see the README.

Settings

Open the extension’s options page to change:

  • Hidden tags — one tag slug per line (default vibecoding).
  • Muted words — one word or phrase per line, matched whole-word and case-insensitively.
  • Mute scope — the whole thread (default) or only the matching comment.

Settings sync across your browsers and apply immediately, no reload needed.

Privacy

No tracking. No network. No nonsense. The only permission requested is storage (for your settings), and the code only ever runs on lobste.rs and lobsters.dev. There is no telemetry, no analytics, and nothing leaves your browser.
